# 教育实践平台 - 启动指南

## 快速启动（推荐：Docker Compose）

项目根目录提供了针对三大操作系统的启动脚本，它们都会执行 `docker compose up -d` 来拉起 MySQL、后端 API、管理端和用户端：

| 系统 | 启动命令 |
| --- | --- |
| macOS | 双击 `start-macos.command` 或在终端执行 `./start-macos.sh`（首次或依赖发生变化时可附加 `--rebuild`） |
| Linux | `./start-linux.sh` （若提示无权限，可使用 `sudo ./start-linux.sh`） |
| Windows | `powershell -ExecutionPolicy Bypass -File .\start-windows.ps1 [-Rebuild]` |

脚本会自动检测 `docker compose`（或独立的 `docker-compose`），并在结束时输出服务访问地址。若不想使用脚本，可直接在项目根目录运行：

```bash
docker compose up -d --build
```

> 需要停掉服务时执行 `docker compose down`（或 `docker-compose down`）。

## 手动启动（开发模式）

1. **启动后端服务（端口 28964）**
   ```bash
   cd backend
   npm install
   npm run dev
   ```

2. **启动管理端前端（端口 18964）**
   ```bash
   cd frontend/admin
   npm install
   npm run dev
   ```

3. **启动用户端前端（端口 8964）**
   ```bash
   cd frontend/user
   npm install
   npm run dev
   ```

## 停止项目

| 系统 | 停止命令 |
| --- | --- |
| macOS | 双击 `stop-macos.command` 或执行 `./stop-macos.sh`（加 `--volumes` 可删除持久化卷） |
| Linux | `./stop-linux.sh`（如需删除卷可加 `--volumes`） |
| Windows | `powershell -ExecutionPolicy Bypass -File .\stop-windows.ps1 [-RemoveVolumes]` |

若使用手动模式，直接在运行后端/前端的终端中按 `Ctrl+C` 结束即可。

## 访问地址

- **用户端前端**: http://localhost:8964
- **管理端前端**: http://localhost:18964
- **后端 API**: http://localhost:28964

## 注意事项

1. **Docker**：请确保本机已安装并启动 Docker（Windows/macOS 可使用 Docker Desktop，Linux 可安装 Docker Engine）。
2. **端口占用**：Docker Compose 会占用 3306、28964、18964、8964，如被占用请先释放。
3. **依赖安装**：仅在手动运行模式下需要在 backend、frontend/admin、frontend/user 目录执行 `npm install`。
4. **PowerShell 执行策略**：如 Windows 无法运行脚本，可先执行：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

## 故障排除

### 端口被占用
Docker Compose 启动失败并提示端口占用时，可先查找并结束相关进程：
```bash
netstat -ano | findstr :28964
netstat -ano | findstr :18964
netstat -ano | findstr :8964
# 关闭指定 PID（Windows 示例）
taskkill /F /PID <PID>
```

### 依赖安装失败
仅在手动模式下可能遇到，常见处理方式：
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 数据库连接问题
若选择手动部署 MySQL，请确认 `backend/.env` 中的 `DATABASE_URL` 指向可用的数据库，并保持数据表结构通过 Prisma 同步。
